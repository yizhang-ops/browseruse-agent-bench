browseruse-bench 架构文档

内部文档 v1.x 最后更新：2026-03-26

browseruse-bench 是面向 AI 浏览器 Agent 的一体化评测框架，支持在受控、可复现的条件下，对多个 Agent、多个 Benchmark、多种浏览器和多种模型进行横向对比评测。本文档从系统整体架构出发，逐层讲解各模块职责与数据流向，并说明如何扩展 Agent、Benchmark 和 Browser Provider。

关键设计决策

  • 进程隔离：每个任务由 run.py 以独立子进程调用 scripts/agent_runner.py 执行，依赖冲突（如 browser-use vs skyvern）由各自 venv 隔离。
  • 注册表模式：Agent 和 Browser Backend 均通过装饰器 / 工厂函数注册，新实现无需修改核心调度代码。
  • Schema 契约：AgentResult 使用 extra="forbid"，保证字段精确，防止评测脚本读取 Agent 私有字段。
  • LLM-as-Judge：评测独立于运行，通过 bubench eval 批量调用 OpenAI API 对已有运行结果打分。

§1.1 运行时数据流

一次完整的基准测试由三个独立阶段组成：Run → Eval → Leaderboard。各阶段均可单独执行。

§2 模块详解

本章逐模块拆解各层的内部结构、接口契约与设计决策，覆盖 Agents、Browsers、Benchmarks、Schemas、CLI 和 Leaderboard 六个子模块。

§2.1 Agents 模块

路径：browseruse_bench/agents/

所有 Agent 实现统一继承 BaseAgent，通过 @register_agent 装饰器注册到全局注册表，agent_runner.py 通过名称查找实例。

Agent 所使用的 LLM 模型通过 agent_config["MODEL_ID"] 传入，同一 Agent 可搭配任意模型运行，结果分别落盘在 experiments/…/{Agent}/{MODEL_ID}/ 子目录下，Leaderboard 按 Agent × Model 维度聚合展示。

run_task() 契约

参数类型说明
task_infodict含 task_id、task_text、url 等字段
agent_configdict来自 configs/agents/{agent}/config.yaml，已合并环境变量
task_workspacePath任务输出目录，Agent 可在此写截图、日志等
返回值AgentResult标准化结果，详见 §2.4

BrowserSessionContext 字段

BrowserSessionContext 是 Backend 返回给 Agent 的会话句柄，包含连接 Browser 所需的全部信息（CDP URL、session ID 等），各 Backend 可通过 extra="allow" 附加私有字段。

孤儿会话清理

当 Agent 子进程异常终止时，run.py 会检测 .browser_session_state.json 是否残留，并调用 browsers/orphan_cleanup.py 完成后端侧清理，防止云端浏览器资源泄漏。

§2.5 CLI 模块

路径：browseruse_bench/cli/，入口：bubench 命令（pyproject.toml 定义）

子命令模块职责
bubench runcli/run.py加载数据集、调度任务子进程、汇总结果
bubench evalcli/eval.py读取 result.json，调用 Benchmark 评测逻辑，输出 EvalResult
bubench leaderboardcli/leaderboard.py聚合所有 EvalSummary，生成排行榜 HTML
bubench servercli/server.py启动 FastAPI 服务，提供排行榜 Web UI 和 API
bubench servicecli/service.py管理 systemd 服务（install/start/stop/status）
bubench skillscli/skills.py安装内置开发者 skills 包

scripts/ 目录下的 run.py、eval.py 等是对应 CLI 子命令的薄包装，供 uv run scripts/xxx.py 方式调用。scripts/agent_runner.py 是每个任务子进程的真正入口，不对外暴露为 CLI。

§2.6 Leaderboard 模块

路径：browseruse_bench/leaderboard/

generator.py 扫描 experiments/ 目录下所有 *_eval_summary.json，聚合各 Agent × Benchmark × Model 的指标，输出交互式 HTML 文件。server.py 用 FastAPI 将 HTML 和原始 JSON 以 HTTP 服务形式暴露。

核心指标

指标含义
Success Rate任务成功率（LLM-as-Judge 打分）
Avg Steps成功任务平均步数
Avg E2E Latency端到端延迟（秒）
Avg Tokens成功任务平均 Token 消耗
Avg Cost成功任务平均费用（USD）

代码骨架

# browseruse_bench/agents/my_agent.py
from pathlib import Path
from typing import Any
from browseruse_bench.agents.base import BaseAgent
from browseruse_bench.agents.registry import register_agent
from browseruse_bench.schemas import AgentResult

@register_agent
class MyAgent(BaseAgent):
    name = "my-agent"                       # 唯一标识，对应 --agent 参数

    def prepare(self, agent_config: dict[str, Any]) -> None:
        # 延迟导入、环境检查等初始化逻辑（可选）
        pass

    def run_task(
        self,
        task_info: dict[str, Any],
        agent_config: dict[str, Any],
        task_workspace: Path,
    ) -> AgentResult:
        # 1. 从 agent_config 读取 BROWSER_ID
        browser_id = agent_config.get("BROWSER_ID", "Chrome-Local")

        # 2. 使用 open_browser_session 获取浏览器会话
        from browseruse_bench.browsers.manager import open_browser_session
        with open_browser_session(browser_id, self.name, agent_config) as session:
            cdp_url = session.cdp_url
            # ... 执行任务，截图 ...

        # 3. 返回标准化的 AgentResult
        from datetime import datetime, timezone
        from browseruse_bench.schemas.agent_result import AgentMetrics, AgentUsage
        return AgentResult(
            task_id=task_info["task_id"],
            task=task_info.get("task_text", ""),
            timestamp=datetime.now(timezone.utc),
            env_status="success",
            agent_done="done",
            agent_success=True,
            answer="...",
            model_id=agent_config.get("MODEL_ID", ""),
            browser_id=browser_id,
            metrics=AgentMetrics(end_to_end_ms=1000, steps=5),
        )

from browseruse_bench.agents.my_agent import MyAgent  # noqa: F401

在 config.yaml 的 agents 节点下添加：

# browseruse_bench/browsers/providers/my_provider.py
from typing import Any, Dict
from browseruse_bench.browsers.base import BrowserBackend
from browseruse_bench.browsers.types import BrowserSessionContext

class MyProviderBackend(BrowserBackend):
    def open(self, agent_name: str, agent_config: Dict[str, Any]) -> BrowserSessionContext:
        # 启动/连接远程浏览器，获取 CDP URL
        cdp_url = "ws://..."
        return BrowserSessionContext(
            backend_id=self.backend_id,
            cdp_url=cdp_url,
        )

    def close(self, session_context: BrowserSessionContext) -> None:
        # 释放浏览器资源
        pass

在 browsers/registry.py 的 _register_default_backends() 中添加：

register_backend("my-browser", lambda: MyProviderBackend("my-browser"))

Agent 的 config.yaml 中设置 BROWSER_ID: my-browser 即可使用。
